# モバイルオーダーシステム

出店用のモバイルオーダーシステムです。お客様が注文し、店側がリアルタイムで確認・管理できます。

## 機能

### お客様用画面 (index.html)
- スマホ最適化されたメニュー画面
- 商品の数量選択
- カート機能
- 注文番号の発行

### 店側管理画面 (admin.html)
- **管理者ログイン機能**（パスワード: 12345678）
- 全オーダーの一覧表示
- ステータス管理（待機中→調理中→完成→完了）
- フィルタ機能
- 自動更新（3秒ごと）
- 統計表示
- **本日の集計機能**（テキスト出力・コピー）
- **CSV出力機能**（Excel対応）
- **メニュー編集機能**（商品の追加・編集・削除）

## GitHub Pagesでの使い方

### 1. リポジトリにファイルをアップロード
以下のファイルをリポジトリにアップロード：
- `index.html` (お客様用)
- `admin.html` (店側管理用)

### 2. GitHub Pagesを有効化
1. リポジトリの Settings → Pages
2. Source を "main" ブランチに設定
3. Save

### 3. アクセス
- お客様用: `https://[ユーザー名].github.io/[リポジトリ名]/`
- 店側管理: `https://[ユーザー名].github.io/[リポジトリ名]/admin.html`

## 管理者機能の使い方

### ログイン
- 管理画面にアクセスするとログイン画面が表示されます
- パスワード: **12345678**
- ログイン後はセッション中ログイン状態が維持されます

### 集計表示
1. 「📊 集計」ボタンをタップ
2. 本日の売上集計が表示されます
   - 注文件数（総数、待機中、調理中、完成、完了）
   - 商品別販売数と売上
   - 総売上金額
3. 「📋 テキストをコピー」でテキスト形式をコピー可能

### CSV出力
1. 「📥 CSV」ボタンをタップ
2. 本日のオーダーデータがCSV形式でダウンロードされます
3. ファイル名: `order_YYYYMMDD.csv`
4. Excel等で開けます（UTF-8 BOM付き）

### メニュー編集
1. 「📝 メニュー」ボタンをタップ
2. メニュー編集画面が表示されます

**既存メニューの編集:**
- 商品名と価格を直接変更できます
- 「削除」ボタンで商品を削除できます

**新しいメニューの追加:**
1. 「新しいメニューを追加」セクションでカテゴリ（フード/ドリンク）を選択
2. 商品名と価格を入力
3. 「➕ 追加」ボタンをタップ

**保存:**
- 「💾 保存して反映」ボタンで変更を確定
- お客様画面に即座に反映されます
- メニューデータはLocalStorageに保存されます

### CSVの内容
- 注文番号
- 日時
- ステータス
- 商品名
- 数量
- 単価
- 小計
- 合計

## データの保存について

このシステムはブラウザのLocalStorageを使用しています。

**重要な注意点:**
- 同じブラウザ・同じドメインでデータが共有されます
- ブラウザのキャッシュをクリアするとデータが消えます
- 複数の端末間でのデータ同期は行われません

### 運用方法

**推奨構成:**
1. **お客様用端末**: お客様のスマホでindex.htmlにアクセス
2. **店側管理端末**: 店側のタブレットやスマホでadmin.htmlを常に開いておく

**注意事項:**
- 店側の端末は同じブラウザで開き続けてください
- 複数の店側端末で管理する場合は、それぞれの端末で独立したデータになります
- 営業終了時に必ずCSV出力して記録を保存してください

## セキュリティ

### パスワード変更方法
admin.htmlの以下の行を編集：
```javascript
const ADMIN_PASSWORD = '12345678';  // ← ここを変更
```

**推奨事項:**
- 本番運用前に必ずパスワードを変更してください
- 8文字以上の複雑なパスワードを設定してください

## メニューのカスタマイズ

**推奨方法: 管理画面から編集**
- 管理画面の「📝 メニュー」ボタンから簡単に編集できます
- コードを直接編集する必要はありません

**手動でカスタマイズする場合:**
LocalStorageの`menuItems`キーを編集することも可能です（上級者向け）
